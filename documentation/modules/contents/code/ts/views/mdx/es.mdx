<SubTitle>Antes de Empezar</SubTitle>
Este tutorial busca el flujo principal de desarrollo con **BeyondJS** y demostrar de manera clara los beneficios que este ofrece. El propósito de este tutorial es que los usuarios/programadores puedan aprender mientras practican, por tanto, los conceptos que definen la estructura
de **BeyondJS** son abordados de forma general, focalizando en hacer un ejercicio rápido, conciso y claro. Para mayor entendimiento de los algunos conceptos, es necesario acceder a las secciones correspondientes en la documentación.

El tutorial se divide de la siguiente forma:

Instalar BeyondJS
Dashboard.
Creación de primer módulo.
Integración con el backend.
Navegación.

Sugerimos que este sea completado, para demostrar cómo en poco tiempo logras integrar código de cliente y servidor, utilizando la misma estructura JavaScript.

Antes de empezar, recomendamos la revisión de los siguientes conceptos para que los tengas presente a medida que avanzas

**Bundle**: Representa un conjunto de archivos fuentes que generan un único archivo final, el cual es incluido en el navegador.
**Module**: Representa uno o múltiples bundles.

<Title>Empezando</Title>

Lo primero que debe hacerse es crear el proyecto, para ello usaremos el paquete <InlineCode>@beyond-js/create-project</InlineCode> se encarga de instalar BeyondJS de manera local y generar la estructura inicial de nuestro proyecto.

<Code>npx @beyond-js/create-project --name @test/login --type web-backend --container beyond</Code>

BeyondJS está realizado para funcionar como dependencia global y correr varios proyectos en simultáneo, puedes aprender más sobre esto <Link to="/docs/quick-start">aqui</Link>

En el comando anterior, se solicitó crear un proyecto de tipo web con backend y se le agregó como scope <InlineCode>testing</InlineCode> y como nombre <InlineCode>login</InlineCode>.

El scope y el nombre, son datos que permiten identificar el proyecto y que lo dejan listo para ser publicado como paquete npm o para importarlo desde otro proyecto. Estas configuraciones pueden hacerse de forma manual, el dashboard sólo simplifica el proceso. Puedes ver la explicación de ello en <Link to="/project/config">la sección de Configuración de proyecto.</Link>

Al finalizar el comando, podemos validar la estructura creada adentro del directorio, es la siguiente:

**beyond.json:** Archivo para manejar el servidor de desarrollo, gestionado por BeyondJS.
**package.json:** Archivo de dependencias del proyecto
**projects.json:** Archivo de configuración de proyectos, permite a BeyondJS identificar los proyectos a levantar.
**login:** Carpeta del proyecto.

// SUB List
**project.json**: Contiene la configuración básica del proyecto (puede variar según el tipo y las necesidades del mismo).
**/modules:** Carpeta en donde se crean los módulos del proyecto por defecto. Este valor es configurable, puedes leer más de ello en la sección de <Link to="/docs/projects/json">Configuración de Proyectos.</Link>
**template:** Esta carpeta contiene todos los elementos necesarios para definir el template general del proyecto.

// SUB SUB LIST
**template.json:** Archivo que permite realizar ajustes en la configuración del template del proyecto.

Puedes aprender más acerca de los tipos de proyecto disponibles en la sección de <Link to="/docs/quick-start">Inicio Rápido.</Link>

A continuación, debemos iniciar beyond, para ello ejecutamos el siguiente comando:

<Code>npm start</Code>

Luego de ejecutar el comando, puede verse un mensaje que indica que el Dashboard está disponible en <ELink to="https://workspace.beyondjs.com/">https://workspace.beyondjs.com/</ELink> lo navegamos y , podremos ver la interfaz del Dashboard con el board del listado de proyectos, donde aparece el proyecto creado con el link para navegarlo debajo del titulo, si hacemos click en el, la pantalla se dividirá en dos y podremos navegar el proyecto desde el dashboard.
