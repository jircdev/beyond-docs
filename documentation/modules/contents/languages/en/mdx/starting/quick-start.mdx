#### Prerequisites

To understand this documentation, you should keep in mind the following points:

-   Having Node ^14.17.4 installed.
-   Being familiar with dependency managers such as npm or yarn.

<BeyondJS /> can be installed through two commands:

Global installation with npm: npm i -g beyond.
Installation and project creation with npx: npx @beyond-js/create-package.

<Title tag="h3" text="Install and create a project" />
You can install and create a project at the same time through the following npx command:

```
npx @beyond/create-package
```

The command is interactive and will prompt for multiple options, in which we will select the following:
This is the ideal command if you do not have <BeyondJS/> installed and want to start a new project at the same time.
It performs global installation and proceeds to automatically create a package in the directory where it is executed.

When executed, it will ask for the package data, which you can view in this documentation
in <Link href="/docs/quick-start#create-package">the create package section.</Link>

<Title tag="h3" text="Global installation" />

You can install <BeyondJS/> globally using the following command:

<Code>npm i -g beyond</Code>

The previous command installs <BeyondJS/> globally and enables all CLI commands. You can read more about them in the <Link href="/docs/cli">CLI commands section</Link>.

Next, you can <Link href="/docs/quick-start#create-package">create a package from scratch</Link> or run <BeyondJS/> on a folder with existing projects.

<Title id="create-package" tag="h3" text="Creating a package" />
You can reach this step if you ran the npx command to create a project or having <BeyondJS /> installed through the following
command:

<Code>beyond add package</Code>

When running it, the interactive console will ask you for the following information:

specifier: which corresponds to the package name.
title: A commercial and readable title.
type: Corresponds to the type of package you want to create. <BeyondJS/> has several base templates that include the most common front libraries or work structures for back-end packages. You can read more about it in <Link href="/docs/packages/create">the package creation section</Link>.
After entering the data, <BeyondJS /> will finish the installation process and you will be ready to start programming, you just need to initialize the DevServer with the following command:

<Code>beyond run --workspace 4000</Code>

<Title tag="h3" text="Instalación global" />

Este comando realiza una instalación global que permite ejecutar el <Link href="/dev-server">DevServer</Link> de

<BeyondJS /> en el directorio que se desee y habilita la lista de comandos disponibles en el CLI.

> Para ver la lista de comandos de <BeyondJS/> puedes ir a <Link href="/docs/cli"> la sección del CLI</Link>

<Title tag="h3" text="Initialize BeyondJS" />

This command initializes BeyondJS and allows you to navigate to your
project at <ELink href="http://localhost:6530">http://localhost:6500</ELink>.

> The `6530` port is defined by default in the project distribution and can be easily modified in the package.json.
> You can read in detail about it in <Link href="/distributions"> the distributions section</Link>.

<Title tag="h3" text="Create a module" />

You can create a new module in <BeyondJS/> using the CLI by running the following command from the project
folder (where the `package.json` file is located):

<Code>beyond add module</Code>

> The modules in <BeyondJS/> are handled through a `module.json` file with a minimal and simple configuration, and
> the previous command automatically generates the file. You can get more information on how to handle modules in
> the <Link href="/beyond/modules/creation">corresponding section of the documentation</Link>
