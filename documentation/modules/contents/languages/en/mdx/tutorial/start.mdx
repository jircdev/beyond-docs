<Title tag="h4">Antes de Empezar</Title>

> Este tutorial busca el flujo principal de desarrollo con **BeyondJS** y

<Title tag="h3" text="Prerequisitos" />

Para ejecutar el presente tutorial se requiere:

-   Tener instalado Node.js 16.x
-   tener conocimientos básicos sobre la consola de comandos.

Adicionalmente, se recomienda usar <ELink href="https://code.visualstudio.com/">vscode</ELink> para
poder seguir el tutorial paso a paso. Sin embargo se puede utilizar cualquier otro editor.

<Title tag="h2" text="Empezando" />

Lo primero que realizaremos es crear nuestro proyecto. Para ello debemos abrir la consola de comandos
en la carpeta en la que deseemos y ejecutar el siguiente comando:

<Code>npx @beyond-js/create-package</Code>

Este comando es interactivo y solicitará algunos datos para definir nuestro paquete o proyecto.
Los datos que pide son los siguientes:

-   Package specifier: Corresponde al nombre de nuestro paquete, para efectos de este tutorial
    nuestro paquete se llamará: 'beyond-web-tutorial'.
-   Package type: Corresponde al tipo de proyecto a crear, seleccionaremos `web-backend` que contiene
    los paquetes para implementar una conexión entre un servicio node y nuestro proyecto web por medio de websockets,
    como vermos más adelante.
    ondJS de manera local y generar la estructura inicial de nuestro proyecto.
-   Preguntará si deseamos instalar las dependencias, seleccionamos que sí.

¡Listo! nuestro proyecto se ha creado. A continuación abriremos nuestro editor de texto en el directorio
donde ejecutamos el comando

<Code>code .</Code>

Al abrirlo podremos ver que tenemos la siguiente estructura:

-   **beyond.json:** Archivo para manejar el servidor de desarrollo, gestionado por BeyondJS.
-   **packages.json:** Este archivo registra los proyectos a los que el DevServer tiene acceso,
    generalmente es gestionado de forma directa por <BeyondJS/>.
-   **beyond-js-tutorial**: La carpeta de nuestro paquete y proyecto.

    -   **package.json**: Archivo del paquete javascript.
    -   **/modules:** Carpeta en donde se crean los módulos del proyecto por defecto. Este valor es configurable, puedes leer más de ello en la sección de <Link href="/docs/projects/json">Configuración de Proyectos.</Link>
    -   **template:** Esta carpeta contiene todos los elementos necesarios para definir el template general del proyecto.

        -   **template.json:** Archivo que permite realizar ajustes en la configuración del template del proyecto.

> Si deseas puedes conocer las diferencias sobre los tipos de proyectos
> en <Link href="/docs/packages/types"> la sección correspondiente de la documentación.</Link>

A continuación, debemos iniciar <BeyondJS/>, para ello ejecutamos el siguiente comando:

<Code language="shel">beyond run</Code>

Al ejecutarlo, debe aparecer el siguiente mensaje en la consola de comandos:

```
Follow the link to manage your packages with the workspace:
https://workspace.beyondjs.com?port=4000

```

¡Listo! Ya podemos acceder a nuestro proyecto en <ELink href="http://localhost:6500/">http://localhost:6500/</ELink>
