<DocHeader id="intro" pretitle="Fundamentos" title="Distribuciones" />

Las Distribuciones son una funcionalidad que permite definir el comportamiento
que debe tener el DevServer de BeyondJS en el entorno de desarrollo y las caracteristicas para el
empaquetado del proyecto durante el proceso de despliegue

> Al crear un nuevo proyecto, este viene con una distribución configurada de forma predeterminada.
> En este documento se describen en detalle todas las opciones de configuración de las distribuciones,
> necesarias para trabajar con diferentes entornos de ejecución o tipos de resolución de módulos.

<Title id="definition" tag="h2" text="Definición" />

Las distribuciones se definen en el archivo `project.json`, en la propiedad `distirbutions`
del entry point `development`.

Un ejemplo de una distribución es:

<Code>
    {` {
    "name": "web",
    "platform": "web",
    "environment": "development",
    "ports": {
        "bundles": 3500
    }
},
`}
</Code>

<Title id="properties" tag="h2" text="Propiedades" />

-   <ObjectProperty type="string">name</ObjectProperty> Nombre identificador de la distribución.
-   <ObjectProperty type="string">platform</ObjectProperty> identificador de la plataforma que utiliza la distribución. Puede
    ver los identificadores de plataformas en la <Link href="/platforms">sección de Plataformas</Link>

-   <ObjectProperty type="string">environment</ObjectProperty> soporta los valores `deployment` y `production`.

    -   <ObjectProperty type="object">ports</ObjectProperty> recibe un objeto con las siguientes propiedades :
    -   <ObjectProperty type="number">bundles</ObjectProperty> puerto en el que el DevServer devolverá los bundles.
    -   <ObjectProperty type="number">http</ObjectProperty> Puerto para conectar debuggers o inspectores de código. Puede
        leer más sobre inspección remota en <Link href="/remote-inspection">sección de inspección remota.</Link>

-   <ObjectProperty type="object">ts</ObjectProperty> Objeto manejado para gestionar configuraciones del compilador typescript,
    actualmente soporta las siguientes propiedades:

    -   <ItemProperty type="string" optional label="compiler">
            acepta los valores `ts` y `transpiler`.
        </ItemProperty>

-   <ItemProperty type="string" optional label="backend">
        Esta propiedad es requerida si el paquete utiliza una conexión con un backend por medio de websockets. Puedes
        leer más acerca de esto en la <Link href="/bridges">sección de Bridges</Link> .
    </ItemProperty>
