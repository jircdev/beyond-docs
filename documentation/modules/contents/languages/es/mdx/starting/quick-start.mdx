### Prerequisitos

Para entender esta documentación, debes tener en cuenta los siguientes puntos:

-   Tener instalado Node ^14.17.4.
-   Estar familiarizado con manejadores de dependencias como npm o yarn.

<BeyondJS /> puede ser instalado mediante dos comandos:

-   Instalación global con npm: npm i -g beyond.
-   Instalación y creación de un proyecto con npx: npx @beyond-js/create-package.

<Title tag="h3" text="Instalar y crear un proyecto" />

Puedes instalar y crear un proyecto al mismo tiempo mediante el siguiente comando de npx:

```
npx @beyond/create-package
```

El comando es interactivo y procederá a solicitar múltiples opciones, en ellas seleccionaremos las siguientes:
Este es el comando ideal si no tienes <BeyondJS/> instalado y deseas empezar con un proyecto nuevo al mismo tiempo.
Realiza la instalación global y procede a crear un paquete automáticamente en el directorio en el que se ejecuta.

Al ejecutarlo, se te pedirá los datos del paquete, los cuales puedes ver en esta documentación
en <Link href="/docs/quick-start#create-package">la sección de crear paquete.</Link>

<Title tag="h3" text="Instalación global" />

Puedes instalar <BeyondJS/> de forma global por medio del siguiente comando:

<Code>npm i -g beyond</Code>

El comando anterior instala <BeyondJS/> de forma global y habilita todos los comandos del CLI. Puedes leer más acerca de ellos en la
sección <Link href="/docs/cli">de comandos de cli</Link>.

A continuación, puedes <Link href="/docs/quick-start#create-package">crear un paquete desde cero</Link> o ejecutar <BeyondJS/> sobre una carpeta con proyectos existentes.

<Title id="create-package" tag="h3" text="Crear un paquete" />

Puedes llegar a este paso si ejecutaste el comando de `npx` para crear un proyecto o teniendo <BeyondJS/> instalado por medio del siguiente
comando:

<Code>beyond add package</Code>

Al ejecutarlo, la consola interactiva te pedirá la siguiente información:

-   specifier: que corresponde al nombre del paquete.
-   title: Un titulo comercial y legible.
-   type: Corresponde al tipo de paquete que quieres crear. <BeyondJS/> cuenta con varios templates bases que incluyen las librerias
    más comunes de front o estructuras de trabajo para paquetes backend. Puedes leer más sobre ello en <Link href="/docs/packages/create">
    la sección de creación de paquetes</Link>.

Luego de ingresar los datos, <BeyondJS /> terminará el proceso de instalación y estarás listo para empezar a programar, sólo necesitas
inicializar el DevServer con el siguiente comando:

<Code>beyond run --workspace 4000</Code>

<Title tag="h3" text="Instalación global" />

Este comando realiza una instalación global que permite ejecutar el <Link href="/dev-server">DevServer</Link> de

<BeyondJS /> en el directorio que se desee y habilita la lista de comandos disponibles en el CLI.

> Para ver la lista de comandos de <BeyondJS/> puedes ir a <Link href="/docs/cli"> la sección del CLI</Link>

<Title tag="h3" text="Ejecución del proyecto" />

Teniendo instalado <BeyondJS/> y habiendo creado el proyecto, sólo necesitas ejecutar el DevServer en la raiz de tu proyecto
por medio del siguiente comando:

<Code>beyond run --workspace 4000</Code>

Este comando inicializa BeyondJS y te permite navegar a tu proyecto en <ELink href="http://localhost:6530"> http://localhost:6500</ELink>.

> El puerto `6530` es definido por defecto en la distribución de proyecto y puede modificarse de forma simple en el `package.json`.
> Puedes leer en detalle sobre ello en <Link href="/distributions"> la sección de distribuciones</Link>.

<Title tag="h3" text="Crear un módulo" />

Puedes crear un nuevo módulo en <BeyondJS/> utilizando el CLI y ejecutando el siguiente comando desde la carpeta del proyecto
(donde se encuentra el archivo `package.json`):

<Code>beyond add module</Code>

> Los módulos en <BeyondJS/> se manejan mediante un archivo `module.json` con una configuración mínima y sencilla, y
> el comando anterior genera automáticamente el archivo. Puedes obtener más información sobre cómo manejar los módulos en
> la <Link href="/beyond/modules/creation">sección correspondiente de la documentación</Link>.
