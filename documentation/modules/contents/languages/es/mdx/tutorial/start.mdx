<Title tag="h2" text="Empezando" />

Lo primero que debe hacerse es crear el proyecto, para ello usaremos el paquete `@beyond-js/create-project` se encarga de instalar BeyondJS de manera local y generar la estructura inicial de nuestro proyecto.

<Code>npx @beyond-js/create-project --name @test/login --type web-backend --container beyond</Code>

BeyondJS está realizado para funcionar como dependencia global y correr varios proyectos en simultáneo, puedes aprender más sobre esto <Link to="/docs/quick-start">aqui</Link>

En el comando anterior, se solicitó crear un proyecto de tipo web con backend y se le agregó como scope ` testing` y como nombre `login`.

El scope y el nombre, son datos que permiten identificar el proyecto y que lo dejan listo para ser publicado como paquete npm o para importarlo desde otro proyecto. Estas configuraciones pueden hacerse de forma manual, el BeyondJS Workspace sólo simplifica el proceso. Puedes ver la explicación de ello en <Link to="/project/config">la sección de Configuración de proyecto.</Link>

Al finalizar el comando, podemos validar la estructura creada adentro del directorio, es la siguiente:

-   **beyond.json:** Archivo para manejar el servidor de desarrollo, gestionado por BeyondJS.
-   **package.json:** Archivo de dependencias del proyecto
-   **projects.json:** Archivo de configuración de proyectos, permite a BeyondJS identificar los proyectos a levantar.
-   **login:** Carpeta del proyecto.

    -   **project.json**: Contiene la configuración básica del proyecto (puede variar según el tipo y las necesidades del mismo).
    -   **/modules:** Carpeta en donde se crean los módulos del proyecto por defecto. Este valor es configurable, puedes leer más de ello en la sección de <Link to="/docs/projects/json">Configuración de Proyectos.</Link>
    -   **template:** Esta carpeta contiene todos los elementos necesarios para definir el template general del proyecto.

        -   **template.json:** Archivo que permite realizar ajustes en la configuración del template del proyecto.

Puedes aprender más acerca de los tipos de proyecto disponibles en la sección de <Link to="/docs/quick-start">Inicio Rápido.</Link>

A continuación, debemos iniciar <BeyondJS/>, para ello ejecutamos el siguiente comando:

<Code>beyond run</Code>

Al ejecutarlo, debe aparecer el siguiente mensaje en la consola de comandos:

```
Follow the link to manage your packages with the workspace:
https://workspace.beyondjs.com?port=4000

```

¡Listo! Ya podemos acceder a nuestro proyecto en <ELink href="http://localhost:6500/">http://localhost:6500/</ELink>

Además, puede verse un mensaje que indica que el BeyondJS Workspace está disponible
en <ELink to="https://workspace.beyondjs.com/">https://workspace.beyondjs.com/</ELink> lo navegamos y ,
podremos ver la interfaz del BeyondJS Workspace con el board del listado de proyectos.
