<Title>Project.json</Title>

Propiedades project.json:

-   **name** string optional: string Es el nombre para identificar el proyecto, sigue el estándar de npm, por tanto, sólo puede contener caracteres seguros para URLs, sin puntos o guiones bajos.

-   **scope** string optional: Al igual que en npm, sirve para agrupar paquetes y como los proyectos pueden ser publicados como paquetes npm, en caso de agregarse un scope, este debe ser único y sólo puede ser manejado por la organización o usuario que lo utiliza. La validación de unicidad del scope es hecha por npm, sólo si el programador intenta publicar su proyecto como paquete.

-   **title** string optional: Titulo descriptivo del proyecto, no debe poseer más de 100 caracteres.

-   **description** string optional: Espacio para explicar de forma resumida cuales son las características del proyecto.

-   **template** <Link to="/glossary#eoc">EOC</Link> optional: Espacio para explicar de forma resumida cuales son las características del proyecto.

-   **layout** <Link to="/glossary#eoc">EOC</Link> optional: Espacio para explicar de forma resumida cuales son las características del proyecto.

-   **params** <Link to="/glossary#eoc">EOC</Link> optional: Objeto que permite agregar parámetros generales que se requieran utilizar en todo el proyecto, tiene la posibilidad de definir valores por entorno. Para entender más, puedes dirigirte a la sección <Link to="/project/params">Parámetros del proyecto.</Link>

-   **modules** <Link to="/glossary#eoc">EOC</Link> optional: Recibe un objeto con una entrada path que permite definir el directorio de los módulos, por defecto es "module"

-   **deployment** <Link to="/glossary#eoc">EOC</Link> optional: Objeto que permite realizar las configuraciones de los entornos de desarrollo, las cuales son definidas por medio de la entrada deployment

-   **Deployment** <Link to="/glossary#eac">EAC</Link> optional: Objeto que permite realizar las configuraciones de los entornos de desarrollo, las cuales son definidas por medio de la entrada distributions. Las distribuciones pueden agregarse desde la funcionalidad del dashboard para ello o de forma manual. Para entender cómo funcionan puedes dirigirte a la <Link to="/project/distributions">Sección de distribuciones.</Link>

-   **libraries** object optional: Define la importación de proyectos a ser usados como librerías en el proyecto definido.

    -   **imports** string optional: Cada entrada representa un proyecto importado, la importación se hace agregando el scope y nombre del proyecto importado.
