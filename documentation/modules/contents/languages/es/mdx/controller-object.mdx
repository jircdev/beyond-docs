<Title>Objeto controller</Title>

<SubTitle>Definición del Controller</SubTitle>

Todo widget debe exponer una clase Controller definida. El objeto Controller es el responsable de la declaración y manejo del web-component que representa al widget y tiene una estructura como la siguiente:

<Code>
{`
import {ReactWidgetController} from '@beyond-js/kernel/react-widget/ts';
import {Page} from './views/page';
export /*bundle*/
class Controller extends ReactWidgetController {
    get Widget() {
        return Page;
    }
}

`}

</Code>

En el código anterior, la clase Controller extiende del objeto ReactWidgetController que es el controlador base disponibilizado para trabajar con React. BeyondJS tiene disponible un controlador para los distintos frameworks de vista. Los objetos son:

-   ReactWidgetController, ReactPageWidgetController.
-   SvelteWidgetController y SveltePageWidgetController.
-   VueWidgetController y VuePageWidgetController.

Todos los controladores manejan la misma interfaz, pero estan a su vez enfocados en integrar y funcionar con el framework de vista que se esté utilizando.

**Propiedades**

-   **Widget** [getter] : Se define como un getter y debe retornar el componente de vista principal.
-   **uri** <Link to="/docs/api/uri">[URIObject]</Link> [optional] : Objeto Uri con toda la información asociada a la url actual.

**Métodos**

-   **createStore** [function] [optional]: Permite agregar lógica para el manejo del Estado del widget, debe retornar un objeto con la interfaz <Link to="/docs/api/IwidgetStore">IWidgetStore</Link>
-   **initialise** [void] [optional]: Permite agregar lógica a ejecutarse previo al montado del componente web en el DOM HTML.
-   **hide** [void] [optional]: Se ejecuta cuando ha sido renderizada y se oculta, esto sucede cuando se navega de una URLl a otra.
-   **show** [void] [optional]: Se ejecuta cada vez que se accede a la URL y la página se muestra en el navegador.
