<BeyondJS /> ofrece una variedad de paquetes independientes para que puedan ser incorporados en el desarrollo según sea
necesario. Esto proporciona una mayor flexibilidad y adaptabilidad en la creación de proyectos, permitiendo elegir las
tecnologías y librerías que mejor se adapten a las necesidades del proyecto. Además, esto permite una mayor
escalabilidad ya que los paquetes pueden ser actualizados o reemplazados de forma independiente. Puedes leer más acerca
de estos paquetes en la sección de Paquetes disponibles.

<Title tag="h3" text="@beyond-js/backend" />

Este paquete implementa un servicio en node que disponibiliza una plataforma backend con conexión de Websocket y permite
la ejecución de los Bridges, una funcionalidad de BeyondJS que permite crear APIs backend y el código cliente para consumirlo
con el tipado de datos incluido.

<Title tag="h3" text="@beyond-js/bee" />

Este paquete esta creado para mejorar la experiencia de desarrollo en Node.

El entorno de ejecución Node.js gestiona los paquetes haciendo busqueda de los mismos en disco. El
Beyond Execution Environment de <BeyondJS/> permite que en su lugar, interactúe contra el
DevServer de <BeyondJS/>, lo que permite codificar con funcionalidades de HMR (Hot Module Replacement)
de forma natural y simple.

Sino tienes algo asi es necesario compilar con nodemon que son soluciones obsoletas.

> Node.js ha agregado funcionalidades experimentales para trabajar con http imports a partir de la
> versión 19. <BeyondJS/> brinda un soporte directo para trabajar con http imports y este paquete permite
> programar con la misma experiencia en versiones anteriores de Node.js

<Title tag="h3" text="@beyond-js/local" />

Este paquete es utilzado para dos cosas.

-   Escucha los cambios desde el Inspector de BeyondJS y hace carga inmediata por medio de HMR de los elementos cambiados.
-   Sirve para proyectos backends. se encarga de revisar el estado de los launchers para validar si estan encendidos o no y levantarlos
    previos a la ejecución de la acción.

-   Launcher: Es un objeto que es instanciado por el engine de Beyond por cada distribución backend o node a fin de que pueda ser llamado por el beyond-js local.
    a fin de poder ser ejecuctado de forma automática el proceso de ejecución de estas distribuciones por medio del bee.

Habilita el comportamiento HMR (Hot Module replacement) en los proyectos realizados con <BeyondJS/>.
Es una dependencia para el entorno de desarrollo.

<Title tag="h3" text="@beyond-js/ssr" />

BeyondJS implementa SSR por medio de un servidor HTTP que dispone tres endpoints.

-   widget
-   page

Permite implementar soluciones de Server Side Rendering en proyectos web. Puedes leer más acerca del
renderizado SSR, las buenas practicas y otras formas de renderizado
en <Link href="/docs/widgets/rendering"> la seccion de Renderizado con Widgets</Link>

<Title tag="h3" text="@beyond-js/widgets" />

Tiene un controlador base

Los proyectos o paquetes Web en <BeyondJS/> funcionan como microfrontends independientes
que pueden contener su propio stack de tecnologías. Estos proyectos son flexibles y fácilmente extensibles,
permitiendo agregar nuevos paquetes de widgets que brinden soporte a librerías de vista o renderizado con pocos pasos simples.

> <BeyondJS /> se enfoca en facilitar la integración de nuevas librerías de renderizado, para que puedas hacerlo de
> forma sencilla. Puedes leer más acerca de cómo agregar motores de renderizado en la sección correspondiente del
> documentación.
