<SubTitle>Antes de Empezar</SubTitle>
Este tutorial busca el flujo principal de desarrollo con <strong>BeyondJS</strong> y demostrar de manera clara los beneficios que este ofrece. El propósito de este tutorial es que los usuarios/programadores puedan aprender mientras practican, por tanto, los conceptos que definen la
estructura de <strong>BeyondJS</strong> son abordados de forma general, focalizando en hacer un ejercicio rápido, conciso y claro. Para mayor entendimiento de los algunos conceptos, es necesario acceder a las secciones correspondientes en la documentación.

El tutorial se divide de la siguiente forma:

Instalar BeyondJS
Dashboard.
Creación de primer módulo.
Integración con el backend.
Navegación.

Sugerimos que este sea completado, para demostrar cómo en poco tiempo logras integrar código de cliente y servidor, utilizando la misma estructura JavaScript.

Antes de empezar, recomendamos la revisión de los siguientes conceptos para que los tengas presente a medida que avanzas

<strong>Bundle</strong>: Representa un conjunto de archivos fuentes que generan un único archivo final, el cual es incluido en el navegador.
<strong>Module</strong>: Representa uno o múltiples bundles.
<strong>Processor</strong>: Representa el procesador, compilador o transpilador de código de la tecnología que se está utilizando. Empezando Lo primero que debe hacerse es crear el proyecto, para ello usaremos el paquete @beyond-js/create-project se encarga de instalar BeyondJS
de manera local y generar la estructura inicial de nuestro proyecto.

BeyondJS está realizado para funcionar como dependencia global y correr varios proyectos en simultáneo, puedes aprender más sobre esto aqui
npx @beyond-js/create-project --type web-backend --name @testing/login
En el comando anterior, se solicitó crear un proyecto de tipo web con backend y se le agregó como scope testing y como nombre login.

Puedes aprender más acerca de los tipos de proyecto disponibles en la sección de Inicio Rapido.
El scope y el nombre, son datos que permiten identificar el proyecto y que lo dejan listo para ser publicado como paquete npm o para importarlo desde otro proyecto. Estas configuraciones pueden hacerse de forma manual, el dashboard sólo simplifica el proceso. Puedes ver la explicación de ello en La seccion de Configuracion de proyecto
Al finalizar el comando, podemos validar la estructura creada adentro del directorio, es la siguiente:

<strong>beyond.json</strong>: Archivo para manejar el servidor de desarrollo, gestionado por BeyondJS.
<strong>package.json</strong>: Archivo de dependencias del proyecto
<strong>projects.json</strong>: Archivo de configuración de proyectos, permite a BeyondJS identificar los proyectos a levantar. login: Carpeta del proyecto.
<strong>project.json</strong>: Contiene la configuración básica del proyecto (puede variar según el tipo y las necesidades del mismo). /modules: Carpeta en donde se crean los módulos del proyecto por defecto. Este valor es configurable, puedes leer más de ello en la sección de Configuracion
de Proyectos
<strong>backend</strong>: it starts the backend platform
<strong>home</strong>: basic page module with a default route
<strong>layouts</strong>: Folder with default layout of the web project template: Esta carpeta contiene todos los elementos necesarios para definir el template general del proyecto.
<strong>template.json</strong>: Archivo que permite realizar ajustes en la configuración del template del proyecto. Luego de haber ejecutado el comando, ya estamos listos para empezar nuestro proyecto, para ello ejecutamos lo siguiente.

> beyond
> Luego de ejecutar el comando puede verse un mensaje que indica que el Dashboard está disponible en http://localhost:4000, lo navegamos y podemos ver la interfaz del Dashboard con el board del listado de proyectos, donde aparece el proyecto creado con el link para navegarlo debajo del titulo, si hacemos click en el, la pantalla se dividirá en dos y podremos navegar el proyecto desde el dashboard.

BeyondJS - Module List view
El puerto del proyecto se define en el archivo project.json, puedes leer más sobre ello en la sección de Configuracion de Proyectos
La funcionalidad de navegación es un feature del dashboard y es opcional, el proyecto puede navegarse de forma independiente en cualquier navegador.
Ahora estamos listos para empezar a trabajar en el primer módulo.
